# Workplace Boundary

職場の圧力パターンを認識し、証拠にもとづいて境界線を守るための agent skills です。

短く言うと、感情的な対立ではなく、証拠、記録、責任範囲、期限、意思決定の明確化によって自分を守るためのツールです。

## このプロジェクトについて

`workplace-boundary` は、防御目的の公開 agent skill パッケージです。複雑な職場状況を、観察可能な行動、記録、責任、意思決定、証拠に分解し、専門的で送信可能な対応文を作ることを目的としています。

このプロジェクトは「相手がどんな属性の人か」ではなく、次の点を見ます。

- 何が言われ、書かれ、割り当てられ、変更され、承認され、記録されたのか。
- どの会議、メール、チケット、資料、評価資料、意思決定ログが事実の基準になるのか。
- 貢献、責任者、期限、リスク、承認、説明責任が曖昧にされていないか。
- 記録を守るための最小限で適切な行動は何か。

## 公開されている skills

### `workplace-pressure-patterns`

観察可能な職場の圧力パターンとリスク経路を特定します。

- 貢献の横取り、可視性の奪取；
- 責任転嫁、責任範囲の漂流；
- owner、期限、意思決定、承認状態の曖昧化；
- 議事録、資料、status update、評価資料の歪み；
- 非公式チャネルでの決定、事前に固定されたナラティブ；
- 継続的な見えない仕事を引き受けさせる圧力。

出力の中心は、行動診断、証拠の強さ、リスク経路、防御への引き継ぎです。

### `workplace-boundary-defense`

診断を、専門的で証拠にもとづく防御文書に変換します。

- 会議後の確認；
- 議事録修正；
- 貢献マップ；
- owner、期限、承認、リスクの確認；
- status update；
- escalation note；
- 評価資料や昇進資料への証拠補足；
- source-of-truth record の修正。

低対立、高証拠の境界防御を目的としており、敵対的、操作的、差別的、報復的な文面は生成しません。

## Agent による対抗シミュレーション

開発過程では、現実的な職場シナリオを内部 agent 対抗シミュレーションで検証しました。より強い圧力、記録の歪み、貢献の横取り、責任転嫁、責任移転に対して、防御側の skill が耐えられるかを確認するためです。

公開されているのは防御側の skills だけです。

内部 Red Team コンポーネント、プロンプト、スクリプト、実行ログ、評価記録、攻撃手順として再利用され得る資料は、この公開リポジトリには含まれていません。

最新の公開版には、非公開の agent レビュー済み職場ノートから抽象化した、出典を含まない防御パターンも含まれています。公開されるのは抽象的な境界防御パターンのみで、元ノート、タイトル、作者、引用、パス、逐条レビュー data、内部テスト資料は含まれません。

## Red Team を公開しない理由

目的は、人が自分を守れるようにすることであり、同僚を抑圧する方法を教えることではありません。

この公開リポジトリに含まれるもの：

- 行動ベースのパターン認識；
- 証拠規律；
- 境界防御；
- 責任者と説明責任の明確化；
- 記録修正；
- 最小限の escalation；
- 属性に依存しない反差別ガードレール。

公開しないもの：

- 他人に責任を押しつける手順；
- より目立たずに貢献を奪う方法；
- 会議や記録を操作する方法；
- 証拠を回避する方法；
- そのまま送信できる攻撃的な職場文面。

## インストール

```bash
cp -R skills/workplace-pressure-patterns ~/.agents/skills/
cp -R skills/workplace-boundary-defense ~/.agents/skills/
```

2 つの skill を一緒に使うことを推奨します。一方が圧力経路を診断し、もう一方が証拠にもとづく防御文を作成します。

## 適切な使い方

使える用途：

- 具体的な会議、メール、チャット、status update、評価資料の分析；
- 感情的な不満を、行動と証拠の問いに変換する；
- 落ち着いた記録修正や確認メッセージを作る；
- 貢献、責任者、期限、承認状態、リスク、source-of-truth record を守る。

使ってはいけない用途：

- 嫌がらせ、差別、報復、属性にもとづく攻撃；
- 国籍、民族、人種、性別、カースト、宗教、アクセント、移民状態などから行動を推測すること；
- 証拠の捏造、貢献の誇張、存在しない承認の示唆；
- 操作、排除、責任転嫁、貢献の横取り、抑圧のための戦術生成。

## 基本原則

属性ではなく、行動を分析する。

職場の境界防御とは、事実、証拠、owner、期限、意思決定、リスク、audience、receipt、source of truth を管理する記録システムです。
